# 大坝缺陷检测项目 - 使用指南

## 一、项目简介

基于 **SegFormer-B2 + MMSegmentation** 的大坝表面缺陷语义分割项目。

- **训练模型**：SegFormer-B2（MiT-B2 backbone + SegformerHead），7类语义分割
- **推理工具**：推理 + 实例级量化 + 可视化，7类→4类重映射，支持裂缝端点桥接

---

## 二、环境配置

### 2.1 基础环境要求

| 项目 | 要求 |
|------|------|
| Python | 3.8 ~ 3.10（推荐 3.9） |
| CUDA | 11.8（推荐） |
| PyTorch | **必须 2.1.2**（不要用 2.5+，mmcv 没有对应预编译 wheel） |
| GPU 显存 | 训练 >= 8GB，推理 >= 4GB |

### 2.2 安装步骤

以下所有命令在项目根目录 `/root/work/DamInspection` 下执行。

#### Step 1：创建虚拟环境

```bash
conda create --prefix /root/work/damseg python=3.9 -y
conda activate /root/work/damseg
```

#### Step 2：安装 PyTorch 2.1.2

```bash
pip install torch==2.1.2 torchvision==0.16.2 --index-url https://download.pytorch.org/whl/cu118
```

> **不要安装 PyTorch 2.5+**，mmcv 2.1.0 没有对应版本的预编译 wheel。

#### Step 3：安装 mmengine

```bash
pip install "mmengine>=0.5.0,<1.0.0"
```

#### Step 4：安装 mmcv（必须使用预编译 wheel）

**mmcv 不能直接 `pip install mmcv`**，会尝试源码编译并报错 `ModuleNotFoundError: No module named 'pkg_resources'`。必须从 OpenMMLab 预编译索引安装：

```bash
pip install mmcv==2.1.0 -f https://download.openmmlab.com/mmcv/dist/cu118/torch2.1/index.html
```

#### Step 5：安装 mmsegmentation 并覆盖自定义文件

本项目在 `mmsegmentation/` 目录下有自定义的 Dataset 和 Transform（115 个文件），但目录不完整，缺少核心文件（如 `decode_head.py`）。**不能直接 `pip install -e .`**，否则会报 `ModuleNotFoundError`。

正确做法：先安装完整 mmseg，再把自定义文件覆盖进去。

```bash
# 1. 安装完整的 mmseg 1.2.2
pip install mmsegmentation==1.2.2

# 2. 把项目中的自定义文件覆盖到 pip 安装的完整包中
cp -r /root/work/DamInspection/mmsegmentation/mmseg/* \
      /root/work/damseg/lib/python3.9/site-packages/mmseg/
```

> **原理**：步骤 1 安装了完整的 mmseg（包含所有核心文件），步骤 2 把自定义文件（`S2DSDeepCrackDataset`、`SegformerHead`、`ClassBalancedRandomCrop` 等）覆盖进去，得到一个**完整 + 包含自定义修改**的 mmseg 包。

> **注意**：覆盖后如果升级了 mmseg 包，需要重新执行步骤 2。

#### Step 6：安装推理与量化脚本的依赖

```bash
pip install "numpy<2" opencv-python scikit-image scipy pyyaml pandas tqdm Pillow scikit-learn ftfy
```

> **`numpy<2` 是必须的**，mmcv 2.1.0 与 NumPy 2.x 不兼容，会报 `_ARRAY_API not found`。
> **`ftfy` 是必须的**，mmsegmentation 的 tokenizer 模块依赖它。

#### Step 7：（可选）安装 COCO RLE 编码支持

```bash
pip install pycocotools
```

#### 一键安装（完整复制版）

```bash
# 1. 创建环境
conda create --prefix /root/work/damseg python=3.9 -y
conda activate /root/work/damseg

# 2. PyTorch
pip install torch==2.1.2 torchvision==0.16.2 --index-url https://download.pytorch.org/whl/cu118

# 3. OpenMMLab
pip install "mmengine>=0.5.0,<1.0.0"
pip install mmcv==2.1.0 -f https://download.openmmlab.com/mmcv/dist/cu118/torch2.1/index.html

# 4. 完整 mmseg + 自定义文件覆盖
pip install mmsegmentation==1.2.2
cp -r /root/work/DamInspection/mmsegmentation/mmseg/* /root/work/damseg/lib/python3.9/site-packages/mmseg/

# 5. 推理依赖
pip install "numpy<2" opencv-python scikit-image scipy pyyaml pandas tqdm Pillow scikit-learn pycocotools ftfy
```

## 三、数据集准备

### 3.1 数据集结构

数据集放在 `/root/work/s2ds_deepcrack/`，结构如下：

```
/root/work/s2ds_deepcrack/
├── train_img/        # 训练图片（RGB .png）
├── train_lab/        # 训练标注（单通道 .png，与图片同名）
├── val_img/          # 验证图片
├── val_lab/          # 验证标注
├── test_img/         # 测试图片
└── test_lab/         # 测试标注
```

### 3.2 标注格式

使用 **S2DSDeepCrackDataset**，标注为单通道 PNG（L 模式），像素值即类别 ID：

| 像素值 | 类别 |
|--------|------|
| 0 | background（背景） |
| 1 | crack（裂缝） |
| 2 | spalling（剥落） |
| 3 | corrosion（腐蚀） |
| 4 | efflorescence（泛碱） |
| 5 | vegetation（植被） |
| 6 | control_point（控制点） |
| 255 | ignore（不参与 loss 和评估） |

### 3.3 数据集检查

```bash
cd /root/work/DamInspection

python segformer_util/check_data.py \
    --root /root/work/s2ds_deepcrack \
    --datasets . \
    --splits train val test \
    --sample-files 60 \
    --out label_report.json
```

> **注意**：`--datasets .` 表示直接在 `--root` 下查找 `train_lab` 等目录，不要省略。

### 3.4 清理损坏图片

训练前建议批量检查并清理损坏的 PNG 文件，否则 DataLoader 会崩溃：

```bash
python -c "
import cv2, os
for split in ['train', 'val', 'test']:
    img_dir = f'/root/work/s2ds_deepcrack/{split}_img'
    lab_dir = f'/root/work/s2ds_deepcrack/{split}_lab'
    bad = []
    for f in sorted(os.listdir(img_dir)):
        if not f.endswith('.png'): continue
        img = cv2.imread(os.path.join(img_dir, f))
        if img is None:
            bad.append(f)
            os.remove(os.path.join(img_dir, f))
            lab = os.path.join(lab_dir, f)
            if os.path.exists(lab): os.remove(lab)
    if bad:
        print(f'[{split}] removed {len(bad)} corrupted: {bad}')
    else:
        print(f'[{split}] all OK')
"
```

## 六、常见问题排查

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `No module named 'pkg_resources'` | mmcv 源码编译失败 | 用预编译 wheel 安装 mmcv（见 Step 4） |
| `_ARRAY_API not found` | NumPy 2.x 与 mmcv 不兼容 | `pip install "numpy<2"` |
| `No module named 'ftfy'` | 缺少 ftfy 依赖 | `pip install ftfy` |
| `No module named 'mmseg.models.decode_heads.decode_head'` | 自定义 mmseg 目录不完整 | 先 `pip install mmsegmentation==1.2.2`，再 `cp -r` 覆盖自定义文件（见 Step 5） |
| `No module named 'mmseg.models.backbones.vit'` | 同上 | 同上 |
| `failed to load image: xxx.png` | 数据集中有损坏图片 | 用 3.4 节脚本批量清理 |
| `CUDA out of memory` | GPU 显存不足 | 减小 config 中的 `batch_size` |
| 中文显示为 `????` | 缺少中文字体 | `pip install Pillow`，或将字体放入 `fonts/` 目录 |
| `Config file not found` | 配置路径错误 | 检查 `model.mmseg_config` 路径 |
| `Checkpoint not found` | 权重路径错误 | 检查 `model.checkpoint` 路径 |
| bash 报 `No such file or directory` | pip 版本约束被 bash 解析 | 给版本约束加引号：`pip install "pkg>=1.0,<2.0"` |
